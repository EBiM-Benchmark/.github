# Security policy

## Supported versions

The EBiM benchmark currently has no released version. The `main` branch is the only supported target. Once v1.0 is released, the latest minor will be supported.

## Reporting a vulnerability

**Do not open a public GitHub issue for security reports.**

Email: iros.ebim@gmail.com

If your report concerns code that runs on physical robots — including the FR3 Duo control packages, leader/follower teleoperation, or the evaluation harness invoked on real hardware — mark the subject line `[ROBOT-SAFETY]`. We treat any issue that could cause unintended motion, collision, or unsafe end-effector force as critical and respond within 72 hours.

We use GitHub's private vulnerability reporting on public repos (Security tab → Report a vulnerability) as a secondary channel.

## Scope

- All repositories under github.com/EBiM-Benchmark
- The leaderboard backend (URL TBD)
- The evaluation submission pipeline
